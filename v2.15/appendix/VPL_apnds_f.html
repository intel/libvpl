CQP HRD Mode Encoding

CQP HRD Mode Encoding#

The application can configure an AVC encoder to work in CQP rate control mode with HRD model parameters. Intel® VPL will place HRD information to SPS/VUI and choose the appropriate profile/level. It’s the responsibility of the application to provide per-frame QP, track HRD conformance, and insert required SEI messages to the bitstream.

The following example shows how to enable CQP HRD mode. The application should set RateControlMethod to CQP, mfxExtCodingOption::VuiNalHrdParameters to ON, mfxExtCodingOption::NalHrdConformance to OFF, and set rate control parameters similar to CBR or VBR modes (instead of QPI, QPP, and QPB). Intel® VPL will choose CBR or VBR HRD mode based on the MaxKbps parameter. If MaxKbps is set to zero, Intel® VPL will use CBR HRD model (write cbr_flag = 1 to VUI), otherwise the VBR model will be used (and cbr_flag = 0 is written to VUI).

Note

For CQP, if implementation does not support individual QPI, QPP and QPB parameters, then QPI parameter should be used as a QP parameter across all frames.
